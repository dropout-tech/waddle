# Product

## Register

product

## Users

主要為一個人使用為主、但要支持各種角色：個人 SOHO 與在家工作者、重度生產力愛好者（熟悉 Notion / Linear / Todoist）、學生與創意工作者（讀書、寫作、設計類）、以及未來會發展成小型團隊遠距協作。

使用情境跨越早晨啟動、深度工作番茄鐘、會議銜接、傍晚回顧。多數人在桌機長時間使用，手機上多為快速 capture、看今天有什麼、進入專注模式。Slogan「慢慢搖擺，把事情做完」是核心情緒——不催促、不焦慮、有節奏。

長期路線會擴展至小團隊共享 workspace 與進度同步，但現階段所有 UX 決定以「一人」為預設使用者。

## Product Purpose

Waddle 把三件原本散在不同 app 的事——任務管理、時間區塊排程、日記反思——整合到一個三欄式工作面板。讓使用者一邊規劃今日、一邊把任務拖上時間軸、結束時記下感受，全在同一個地方完成。

成功的樣子：使用者一天打開三次以上，並且不會因為 UI 太重而抗拒打開；願意把它當成「自己的數位手帳」而不是「另一個工作工具」。

## North Star — 把自己當一間公司經營

短期我們是手帳，長期我們是「個人作為企業」的成長 agent。

主流趨勢是把 AI agent 導入企業；Waddle 反過來，把企業級的回顧與決策能力帶給個人。日復一日累積下來的任務、時間區塊、日記，最終都不只是紀錄，而是 agent 可以**回顧、回溯、覆盤**的素材——使用者能用對話問它「我這個月卡在哪」、「我讀書的節奏可以怎麼優化」，並且可以切換不同人格（專案管理專家、讀書科學家、教練…）來面對不同情境的成長。

**為什麼這件事重要**：有人說人生的中點是 18 歲，之後愈過愈快。本質上是因為腦袋的快取有限，內容多了記不下來，於是時間感被壓縮。很多人對此感到惶恐。Waddle 想做的，是當大家的**外掛大腦快取**——讓過去不流走、讓回顧變容易、讓成長有跡可循。

**對產品決策的意義**：每一個資料模型、每一個 UX 取捨，都要問一句「這份資料未來能不能被 agent 拿來覆盤？格式有沒有保留足夠的上下文讓未來的自己（或 agent）讀懂？」紀錄不是終點，回顧才是。

## Brand Personality

溫柔、不負擔、有陶瓷感。

- **溫柔**：UI 不催促、不警告、不紅色化任何負面狀態；用詞像朋友而非系統。
- **不負擔**：視覺密度低，留白多；功能再多，每一屏看起來都「還可以再加一個」而不是「太擠了」。
- **有陶瓷感**：表面有溫度、有手感、有重量；不是塑膠、不是玻璃、不是金屬。配色像陶器釉色（赤陶、鼠尾草、玫瑰），邊緣帶手繪感的不完美。

吉祥物是企鵝 Waddle——搖擺、慢、可愛但不幼稚。

## Anti-references

**最重要：不要像冷冰冰的企業 SaaS**——具體點名 Linear、Asana、Jira。具體禁區：

- 深色高對比預設（白底黑字也比黑底白字溫柔）
- 藍紫主色 / Indigo / Slate 配色
- 鋒利的 90° 直角卡片、薄細邊框、緊密 grid
- 「Productivity 訓話」的 UX 文案（"You're behind on 3 tasks"、"Crush your goals"）
- KPI 滿屏的 dashboard 即視感
- AI 工具的 neon gradient / 深色 + 紫藍漸層

也避免：太幼稚的可愛風（Bear、Streaks 那種太 iOS 原生的卡通化）、Material Design 的 FAB + ripple、極簡到冷感的瑞士設計。

## Design Principles

1. **手感優先於效率**。功能設計不以「點擊次數最少」為唯一指標；多一點手寫感、一點動畫的猶豫、一點不完美，比快但無感更重要。
2. **背景化的工具感**。Waddle 是手帳的助手，不是主角。介面要能「淡出」讓內容（任務、行程、日記）成為視覺焦點。
3. **不警告、不催促**。沒有紅色驚嘆號、沒有「逾期」紅字轟炸、沒有 streak 斷掉的焦慮提示。Urgency 用色彩飽和度與層級表達，不用恐嚇。
4. **手機是補完不是縮小**。手機的 bottom sheet、底欄、bottom drawer 是為手機重新設計的容器，不是桌面 sidebar 的縮版。手勢（swipe、long-press、drag）是手機的一等公民。
5. **示範我們提倡的節奏**。產品名叫 Waddle、Slogan 是「慢慢搖擺」——介面動畫、過場、互動都要體現這個節奏：ease-out-quart、200-400ms、不彈跳、不甩動。

## Accessibility & Inclusion

WCAG AA 為基準，特別關注：

- 顏色不是唯一的資訊載體（urgency 等級除了顏色還有形狀 / 標籤）
- 文字對比度 ≥ 4.5:1，尤其是 warm cream 背景上的 charcoal 文字
- 支援 `prefers-reduced-motion`：所有 ease-out-quart 動畫應在 reduced motion 下變成 opacity-only 或 instant
- 鍵盤導航完整支援（任務管理重度使用者期待 j/k、cmd-K 等）
- 手機觸控目標 ≥ 44×44pt（已有 bottom sheet / 底欄遵守）
